# Network Security Group Configuration (Azure Lab)

## Project Overview
This lab demonstrates how Network Security Groups control network traffic between virtual machines in Microsoft Azure.

---

## Environment
- Azure Virtual Machines
- Virtual Network
- Network Security Group

---

## Tools Used
- Azure Portal
- Command Prompt

---

## Tasks Performed
- Created inbound security rules
- Allowed or blocked network protocols
- Tested connectivity between VMs

---

## Steps Performed

1. Logged into Azure Portal
2. Navigated to Network Security Groups
3. Selected the VM security group
4. Opened inbound rules
5. Created rule to allow or block ICMP traffic
6. Saved rule configuration
7. Tested network communication

---

## Result
Network traffic was successfully controlled using security rules.
